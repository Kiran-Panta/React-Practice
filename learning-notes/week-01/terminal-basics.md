# Terminal Basics Notes - Day 2

Today I learned many terminal commands. Here are my notes:

## pwd
This command shows the present working directory, meaning where I am now in the file system.
Example: I used pwd when I was in React-Practice folder, it showed /d/React-js(Practice)

## cd
Change directory, to go to another folder.
Example: cd learning-notes — I used this to go into the learning-notes folder I created.

## ls
List the contents of the current directory.
Example: ls — showed me the files and folders in React-Practice.

## mkdir
Make directory, to create new folders.
Example: mkdir React-Practice — this I used to create the main project folder.

## touch
Create a new empty file.
Example: touch README.md — but actually I created it with content, but touch can make empty files.

## rm
Remove files or directories.
Example: rm oldfile.txt — but I didn't use it today, but learned it's dangerous.

## cp
Copy files.
Example: cp source.txt dest.txt — not used today.

## mv
Move or rename files.
Example: mv oldname.txt newname.txt — not used.

## clear
Clear the terminal screen.
Example: clear — I used this when screen got messy.

## Things that confused me today
- rm deletes permanently, no recycle bin like in Windows. I was scared to use it.
- cd .. means go back to parent directory. At first I thought .. was a file.

This was helpful day. I practiced all these commands.

# Week 6 Reflection — Kiran Panta

## What I Learned This Week

React Router made my app feel like a real website! Before, it was just one page, but now I have multiple pages with navigation.

### Key Concepts:
- **BrowserRouter**: Provides routing context to the entire app
- **Routes & Route**: Renders the first matching route component
- **Link**: For navigation without page refresh
- **useParams**: Gets dynamic values from URL
- **useNavigate**: Programmatic navigation
- **useLocation**: Gets current route information

### API Fetching:
Learned how to fetch data from external APIs using fetch() and async/await. Also learned about loading states, error handling, and caching with useRef.

### Form Validation:
Built a comprehensive form with validation on blur (better UX), password strength meter, and success notifications. Learned about controlled components and single state object pattern.

## Challenges Faced
- Understanding when to use Link vs useNavigate
- Implementing proper form validation without being too annoying
- Caching API responses correctly with useRef

## What I Enjoyed
- Seeing the app become multi-page and professional
- Building the weather widget with real API data
- Creating the portfolio page that showcases everything I built

## Goals for Next Week

- Maybe try some backend integration
- Continue building more complex applications

This week was amazing! React Router really transformed my simple app into something that feels like a real web application.
